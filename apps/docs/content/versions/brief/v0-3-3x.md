# Capacitor

*The people who grow an app should own more of it over time.*
*Track X (canonical): Activity-Investment Marketplace*
*Lineage: v0.3.2-X -> v0.3.3-X*
*Snapshot status: archived prior default canonical*

## The Gap in Token Launches

Launchpads today are mostly liquidity events. Projects launch, trading happens, and value extraction happens fast.

- **pump.fun:** pure attention and trading loops
- **Clanker:** creator fee sharing, but weak connection between actual app activity and ownership growth

What is still missing is a system where:

1. doing useful activity increases your ownership exposure
2. ownership exposure improves your incentive to keep doing useful activity
3. that activity becomes a visible signal for investors deciding where capital should go

## The Core Thesis

Capacitor is a **marketplace for app activity backed by ownership**.

Projects do not just raise attention. They publish measurable work.
Participants (agents or humans) do that work and earn derivative ownership.
Because rewards are tied to a compounding fee pool and lock mechanics, contribution quality and project growth become economically linked.

This creates a flywheel no current launchpad provides:

- activity informs investment
- investment informs activity
- both are visible in one market surface

## Product Definition

Within the broader Capacitor stack, this brief covers the **Emitter** (capital-formation) layer.

### What a project gets

1. Token launch rails
2. Derivative reward token for contributors
3. Task marketplace distribution
4. Public activity telemetry that helps attract both workers and capital

### What a contributor gets

1. A directory of live opportunities across projects
2. Clear task economics (reward, lock, verification, fill rate)
3. Ownership-linked upside for completing useful work

### What an investor gets

1. Real-time project activity visibility
2. Cost-to-growth signals from task spend vs verified outcomes
3. Evidence of whether participation is compounding or stalling

## Marketplace Surface (Yellow Pages for Agentic Work)

Capacitor should expose a public market index where agents and humans can route effort to the highest expected value opportunities.

Each **project card** should include:

- market cap
- volume
- task spend
- verified completions
- active contributors

Each **task card** should include:

- reward terms
- lock terms
- verification method
- completion count
- success rate / rejection rate

This is not just a jobs board. It is a **live economic map** of who is converting capital into real activity.

## Economic Loop

1. Project launches token
2. Trading generates fees
3. A defined share compounds into contributor-backed ownership rails
4. Project publishes measurable tasks
5. Contributors complete tasks and earn derivative rewards
6. Public completion and market signals update continuously
7. More credible activity attracts more contributors and more investor attention
8. Increased volume compounds the contributor pool further

The point is not “tokenize everything.”
The point is to make useful activity and investable signal reinforce each other.

## Why This Is Different

**pump.fun** optimizes for launch velocity.
**Clanker** improves creator economics.
**Capacitor** is built to optimize the activity-investment loop.

No other launchpad currently offers a native system where:

- project activity is directly incentivized through ownership-linked rewards
- contribution outcomes are visible as market intelligence
- investors can evaluate participation quality, not just price action

## MVP Focus

MVP should prove one thing first:

Can we run a credible market where measurable activity, ownership rewards, and investment signal reinforce each other?

To answer that, focus on:

1. launchpad + derivative reward plumbing
2. quantitative task settlement with proof
3. public project/task cards with economic telemetry
4. simple quality routing (entry friction + reputation baseline)

Fragile qualitative governance remains staged R&D.

## Positioning Line

Capacitor is the first launchpad where **doing the work and funding the work are part of the same market loop**.
